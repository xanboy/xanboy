- **Cloud-native services:** identity-aware deployments, secret hygiene, observability-first operations, failure containment

## Engineering posture (how I work)

- **Security boundaries by default:** request validation, auth scopes, secret handling, least privilege
- **Reliability as a feature:** idempotency keys, outbox/queue patterns, backoff, dead-lettering, safe retries
- **Operational clarity:** structured logs, traceable events, minimal “unknown unknowns”
- **Data correctness:** consistent identifiers, currency/amount discipline, reversible adjustments for refunds/cancels
